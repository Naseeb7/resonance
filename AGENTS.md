# AGENTS.md

## Project Context

This project is a premium mobile-first speech therapy PWA called Resonance.

The product experience should feel emotionally supportive, calm, and clinically trustworthy.

Do NOT design this like a gamified children’s learning app.

---

# Core UX Rules

- Prioritize emotional safety over gamification
- Prefer calm and supportive interactions
- Avoid harsh “wrong answer” patterns
- Avoid excessive animations or visual noise
- Use smooth and intentional motion
- Maintain a premium healthcare product feel

---

# UI Direction

Preferred qualities:

- minimal
- warm
- breathable
- premium
- soft motion
- polished mobile interactions

Avoid:

- neon colors
- clutter
- childish visuals
- oversized icons
- aggressive feedback states
- excessive confetti

---

# Technical Rules

- Use TypeScript
- Use functional React components
- Use Tailwind CSS
- Use Framer Motion for transitions
- Keep architecture intentionally simple
- Avoid unnecessary abstractions
- Avoid premature optimization
- No Redux or heavy state libraries
- No backend integration

---

# Component Guidelines

- Components should remain focused and reusable
- Prefer composition over deeply nested logic
- Keep files readable and clean
- Use meaningful component names

---

# Animation Guidelines

Animations should feel:

- smooth
- subtle
- calming
- intentional

Avoid:

- bouncing animations
- excessive scaling
- flashy transitions

---

# Mobile UX Rules

This project is mobile-first.

Prioritize:

- touch ergonomics
- spacing
- safe-area padding
- smooth scrolling
- app-like feel

Desktop support should remain responsive but secondary.

---

# Feedback Language Rules

Do NOT use:

- “Incorrect”
- “Wrong”
- “Failed”

Prefer:

- “Almost there”
- “Getting closer”
- “Try again slowly”
- “That sounded stronger”

Tone should always remain encouraging.

---

# PWA Requirements

The app must:

- be installable
- include manifest support
- support standalone display
- feel app-like on mobile browsers

---

# Code Quality Rules

- Prefer readability over cleverness
- Keep logic simple
- Avoid dead code
- Avoid overengineering
- Keep styling consistent
- Reuse spacing and typography patterns

---

# Important Priority Order

1. Emotional UX quality
2. Interaction polish
3. Mobile responsiveness
4. Animation smoothness
5. Code simplicity
6. Feature completeness

Always optimize for perceived product quality over feature quantity.
